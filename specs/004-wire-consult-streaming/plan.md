# Implementation Plan: Wire Consult Streaming

**Branch**: `001-wire-consult-streaming` | **Date**: 2026-03-18 | **Spec**: [spec.md](./spec.md)

## Summary

Wire the existing home page chat UI to the real NestJS streaming consult endpoint. The recommend flow uses `useChat` from `@ai-sdk/react` with `streamProtocol: 'text'`. A new Next.js route handler at `/api/consult` authenticates via Clerk, then proxies to NestJS `POST /api/v1/consult` using the `HttpClient` abstraction, transforming the NestJS SSE stream into a plain text stream. Recall and add-place flows are unchanged. `AgentResponseBubble` receives a one-line `useEffect` to transition to result phase when streaming content arrives.

### Step 3 тАФ Implement `postStream` in `FetchClient`

**File**: `apps/web/src/api/transports/fetch.transport.ts`

Add `postStream` method. It applies the same auth header logic as `post`, makes the fetch call, checks `res.ok`, and returns the raw `Response` (does NOT call `.json()`). Pattern:

```ts
async postStream(path: string, body: unknown): Promise<Response> {
  const token = await this.getToken()
  const res = await fetch(`${this.baseUrl}${path}`, {
    method: 'POST',
    body: JSON.stringify(body),
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${token}`,
    },
  })
  if (!res.ok) {
    throw new Error(`API error: ${res.status} ${res.statusText}`)
  }
  return res
}
```

### Step 4 тАФ Create `server.ts` (server-side API client)

**File**: `apps/web/src/api/server.ts`

```ts
import { auth } from '@clerk/nextjs/server'
import { createApiClient } from './client'

export function getApiClient() {
  return createApiClient(async () => {
    const { getToken } = await auth()
    const token = await getToken()
    return token ?? ''
  })
}
```

No `'use server'` directive тАФ this is a plain module imported only in server-side code (route handlers).

---

### Step 5 тАФ Create route handler

**File**: `apps/web/src/app/api/consult/route.ts`

Full contract in `contracts/route-handler.md`. Implementation notes:

1. Call `await auth()` from `@clerk/nextjs/server`. If `userId` is null, return `new Response('Unauthorized', { status: 401 })`.
2. Parse request body: `const { messages } = await request.json()`. Extract `query = messages.at(-1)?.content ?? ''`.
3. Call `getApiClient().postStream('/api/v1/consult', { query, stream: true })`.
4. Get the `ReadableStream` from `upstream.body`. If null, return 500.
5. Pipe through a `TransformStream` that parses SSE lines and writes only token content:
   - Lines starting with `data: ` тАФ parse as JSON
   - `type === 'token'` тЖТ write `event.content` to output stream
   - `data: [DONE]` тЖТ close writer
   - Anything else тЖТ skip
6. Return `new Response(readable, { headers: { 'Content-Type': 'text/plain; charset=utf-8' } })`.

The transform runs async in a background task (fire-and-forget `(async () => { ... })()`). Close the writer in a `finally` block to handle upstream disconnects cleanly.

### Step 6 тАФ Extend `AgentResponseBubble`

**File**: `apps/web/src/components/AgentResponseBubble.tsx`

Add one `useEffect` after the existing timer effects:

```ts
useEffect(() => {
  if (content && phase === 'thinking') {
    setPhase('result')
  }
}, [content, phase])
```

This transitions to result phase as soon as the first token arrives. All existing behavior is unchanged when `content` is never provided (recall, add-place, error flows unaffected).
