## Implementation Strategy

1. Complete Phase 1 (T001) — ~1 minute.
2. Complete Phase 2 (T002, T003) — ~3 minutes.
3. Complete Phase 3 (T004 through T009) — ~15 minutes for the code edits + `cap add ios` + `cap sync` + `cap open ios`, plus owner's Xcode manual steps.

### Incremental Delivery

1. Setup + Foundational (~4 min) → foundation ready.
2. US1 (~15 min hands-on + Xcode manual steps) → MVP demo-ready. Commit to `013-capacitor-ios-shell`.
3. US2 (~2 min) → Vercel env confirmed.
4. US3 (~3 min) → backend-untouched verified, Railway CORS verified.
5. US4 (~5 min) → web-untouched verified, browser acceptance verified.
6. Polish (~5 min) → sibling-repo doc update, lint/test pass, final quickstart walkthrough.

Total estimated hands-on implementation time: **~30 minutes**, excluding the owner's Xcode manual steps for signing and device trust.

Not applicable — this is a one-developer feature. Parallelism exists only at the task-within-phase level (T003/T004/T005 can run concurrently in Phase 3; T010–T016 can run concurrently in their respective phases).
